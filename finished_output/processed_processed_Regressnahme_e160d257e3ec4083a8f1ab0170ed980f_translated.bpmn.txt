 Upon receiving a 'Regress reported' event, the system initiates a sequence of tasks to address the situation. First, the 'Check the case' task is triggered, which is directed towards 'ExclusiveGateway2'. This task is designed to investigate the reported regress.

Once the 'Check the case' task is completed, the system proceeds to the 'Close' task, which is also directed towards 'ExclusiveGateway2'. This task signifies the closure of the case after the investigation.

In parallel, the 'Set on re-submission' task is activated due to the 'Regress reported' event. This task is directed towards 'Get money' and is responsible for initiating the re-submission process.

Meanwhile, the 'Book the entrance' task is also triggered, directed towards 'Get money'. This task is responsible for booking the entrance for the re-submission process.

As the 'Regress reported' event unfolds, the 'Proceedings' task is activated, directed towards 'ExclusiveGateway2'. This task is responsible for initiating the proceedings related to the regress.

Additionally, the 'Disclosure of debt collection' task is triggered, directed towards 'Open'. This task is responsible for disclosing the debt collection process related to the regress.

The 'VN payment request' task is also activated, directed towards 'Open'. This task is responsible for handling the payment request related to the regress.

Throughout this process, the 'is Regress authorized?' task is triggered, directed towards 'ExclusiveGateway2'. This task is responsible for determining whether the regress is authorized or not.

The 'justified?' task is also triggered, directed towards 'ExclusiveGateway2'. This task is responsible for determining whether the regress is justified or not.

The 'EventBasedGateway4' task is activated, directed towards 'ExclusiveGateway2' and 'VN contradicts'. This task is responsible for handling events related to the regress.
